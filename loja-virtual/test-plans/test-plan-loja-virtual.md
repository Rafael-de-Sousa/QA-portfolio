# Plano de Testes - Loja Virtual

---

## 1. Introdução

Este documento descreve a estratégia, abordagem, recursos e cronograma dos testes
do sistema Loja Virtual.

---

## 2. Objetivo

Garantir que todas as funcionalidades da loja virtual funcionem conforme esperado,
oferecendo qualidade, segurança e boa experiência ao usuário.

---

## 3. Escopo dos Testes

### 3.1 Funcionalidades Testadas

- Cadastro de usuário  
- Login  
- Carrinho de compras  
- Checkout  
- Pagamento  
- Confirmação de pedido  

### 3.2 Fora do Escopo

- Integrações com gateways reais de pagamento  
- Aplicativos mobile nativos  

---

## 4. Estratégia de Testes

Serão realizados os seguintes tipos de teste:

- Testes Funcionais  
- Testes Exploratórios  
- Testes de Regressão  
- Testes Manuais  
- Testes Automatizados (futuro)  

---

## 5. Ambiente de Testes

- Sistema: Windows 10  
- Navegadores: Chrome, Edge, Firefox  
- Plataforma: Web  

---

## 6. Critérios de Entrada

- Requisitos definidos  
- Ambiente disponível  
- Build estável  

---

## 7. Critérios de Saída

- 95% dos testes executados  
- Bugs críticos resolvidos  
- Aprovação do QA  

---

## 8. Cronograma

| Fase              | Período      |
|-------------------|--------------|
| Planejamento      | 2 dias       |
| Criação de Casos  | 4 dias       |
| Execução          | 5 dias       |
| Regressão         | 2 dias       |
| Finalização       | 1 dia        |

---

## 9. Riscos

| Risco                          | Impacto | Mitigação                     |
|--------------------------------|----------|--------------------------------|
| Instabilidade do sistema       | Alto     | Reforçar testes de regressão   |
| Falta de documentação          | Médio    | Criar casos detalhados         |
| Pouco tempo para testes        | Alto     | Priorizar funcionalidades      |
